# 路由配置与权限守卫 (Router)

## 简介
管理单页应用的路径分发与安全校验。

## 文件结构
```
router/
├── index.ts            # 路由实例创建（user / admin 双模式）
├── constants.ts        # 路径常量与工具函数
├── guard.ts            # 全局路由守卫（Token / 角色 / 管理端动态加载）
├── routes.inner.ts     # 固定路由（登录、错误页、公共页）
├── routes.static.ts    # 用户端静态业务路由（菜单驱动）
├── admin.routes.ts     # 管理端路由定义（独立 chunk）
├── admin.loader.ts     # 管理端路由动态加载器
└── runtime-mode.ts     # 运行时模式判断
```

## 视图路径映射（componentPath）
路由 `componentPath` 对应 `src/views/` 下的文件路径：
- `/user/dashboard/workbench/index.vue` → 用户工作台
- `/user/dashboard/monitor/index.vue` → 用户监控
- `/user/user-center/index.vue` → 个人中心
- `/setting/account/index.vue` → 账户设置
- `/demo/...` → 演示页面
- `/admin/...` → 管理端页面（admin.routes.ts 直接 import）

## 规范
- 路由必须配置 `meta` 属性用于菜单展示。
- 未授权访问必须重定向至登录页。
- 管理端路由独立打包，普通用户不可见。
