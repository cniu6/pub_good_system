# 类型定义与实体模型 (Typings)

## 简介
定义前端全局 TypeScript 类型、接口及 API 数据结构。前端实体定义与后端数据库表结构保持逻辑对齐，遵循严格的命名映射。

## 数据库字段对齐规范
- **后端数据库 (Database)**: 统一使用 `小写+下划线` (如 `user_name`, `is_active`)。
- **前端实体 (Entity)**: 通常转换为 `小驼峰` (如 `userName`, `isActive`)，但在与后端 API 交互时，需确保字段映射正确。
- **严禁**: 在数据库端使用峰驼命名。

## 核心实体字段流证 (Entity)

### 1. 用户实体 (User) - 对应数据库 `users` 表
前端 `User` 接口字段通过 `alova` 拦截器或手动映射与后端下划线字段对应。
- **id** (`number`): 对应 `id`。
- **userName** (`string`): 对应 `username`。
- **nickname** (`string`): 对应 `nickname`。
- **email** (`string`): 对应 `email`。
- **gender** (`0 | 1 | 2`): 对应 `gender` (0:未知, 1:男, 2:女)。
- **money** (`number`): 对应 `money`。
- **score** (`number`): 对应 `score`。
- **level** (`number`): 对应 `level`。
- **role** (`'super' | 'admin' | 'user'`): 对应 `role`。
- **status** (`0 | 1`): 对应 `status` (1:启用, 0:禁用)。
- **createTime** (`number`): 对应 `create_time` (Unix 时间戳)。
- **updateTime** (`number`): 对应 `update_time` (Unix 时间戳)。

### 2. 角色类型 (Role)
- **RoleType**: `'super' | 'admin' | 'user'`。

### 3. 邮件模板 (EmailTemplate)
- **name** (`string`): 模板名称。
- **lang** (`string`): 语言代码。
- **title** (`string`): 标题。
- **content** (`string`): 内容。

## API 数据结构流证 (Api)
- **Service.ResponseResult<T>**: 统一响应格式 `{ code: number, message: string, data: T }`。
- **Api.Login.Info**: 登录接口返回的数据，包含 `user` 实体和 `token`。

## 规范
- 所有的实体定义必须清晰标注其对应的数据库来源字段。
- 复杂类型定义必须存放在 `src/typings/entities/` 目录下。
- 前端字段命名统一采用 **小驼峰 (camelCase)**，与后端的 **下划线 (snake_case)** 在序列化/反序列化时进行转换。
