# 业务视图页面实现 (Views) 深度流证

## 简介
Views 目录是系统的功能展示层，基于 Naive UI 构筑，涵盖了从基础权限到复杂业务场景的完整实现。

## 目录结构

```
views/
├── _builtin/          # 系统内置页面（登录、错误页等）
│   ├── error/         # 403 / 404 / 500
│   ├── forbidden/
│   ├── login/         # 登录 / 注册 / 重置密码
│   └── public-access/ # 公共访问示例
├── admin/             # 管理端页面（独立打包）
│   ├── dashboard/
│   ├── email-templates/

│   ├── settings/
│   └── users/

│   ├── dashboard/     # 仪表盘（workbench / monitor）
│   └── user-center/   # 个人中心
├── setting/           # 设置页面
│   └── account/
├── demo/              # 演示与示例
│   ├── about/
│   ├── cascader/
│   ├── clipboard/

```

## 详细页面流证

### 1. 系统内置页面 (_builtin)
- **Login (登录页)**: 核心表单，集成极验滑动验证、注册、密码重置。
- **Error Pages**: 403 / 404 / 500 错误引导页。

### 2. 用户端页面 (user)
- **Dashboard (仪表盘)**:
    - **Workbench (工作台)**: 聚合展示系统公告、快捷入口、待办事项。
    - **Monitor (监控页)**: 实时系统资源监控，多维度图表。
- **UserCenter (个人中心)**: 展示当前用户的详细资料、修改头像及安全设置。

### 3. 管理端页面 (admin)
- 独立打包，普通用户不可见。
- 包含用户管理、邮件模板、操作日志、系统设置等。

### 4. 设置页面 (setting)
- **Account (账户管理)**: 用户账户相关设置。

### 5. 演示与示例 (demo)
- **Fetch**: 网络请求演示（Token 刷新、数据转换）。
- **Editor**: Markdown 和富文本编辑器。
- **List**: 普通表格、卡片布局、拖拽排序。
- 其他: ECharts、地图、图标、二维码、级联选择器等。

## 技术实现要点
- **组合式函数驱动**: 复杂逻辑（如表格分页、图表自适应）均封装在 `hooks` 中。
- **类型安全**: 所有页面均引入 `typings` 中的实体定义，确保数据渲染无误。
- **响应式适配**: 布局组件自动感应 `isMobile` 状态，动态切换展示模式。
