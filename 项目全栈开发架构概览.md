# 全栈开发平台架构深度解析 (Project Root) 终极流证

## 简介
本项目是一个基于现代技术栈的高性能全栈开发平台。核心设计原则之一是**数据库层与业务层的严格解耦及规范化命名**。

## 数据库核心规范 (Database Strict Convention)
- **命名禁令**: 严禁在数据库层（数据库名、表名、字段名、视图名）使用任何形式的峰驼命名 (CamelCase)。
- **统一风格**: 必须统一使用 `小写字母` 加 `下划线` (`snake_case`)。
- **变量对齐**: 代码中的数据库映射变量建议同步使用下划线风格，以实现端到端的语义一致性。

## 关键技术深度特性
- **数据流闭环**: 
    - 数据库 (snake_case) -> 后端 Model (db tags) -> 前端 API (JSON camelCase)。
    - 系统强制要求数据库层必须为全小写+下划线，以兼容不同数据库系统的敏感性差异。
- **安全性设计**:
    - 基于数据库 `delete_time` 字段实现的逻辑删除，保护数据不被物理抹除。
    - 集成极验 (Geetest) 验证码校验。
    - 基于 JWT 的双 Token (Access & Refresh) 认证机制。

## 开发者手册
1. **数据库修改**: 修改 `backend/internal/db/mysql.go` 中的 `Migrate` 函数，新增字段必须使用小写和下划线。
2. **模型同步**: 更新 `backend/app/models/` 下的结构体，确保 `db` 标签与数据库字段同步。
3. **前端同步**: 更新 `frontend/src/typings/entities/` 下的 TS 定义，并确保 `service/api/` 中的请求函数正确。
