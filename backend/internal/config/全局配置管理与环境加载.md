# 全局配置管理与环境加载 (Config)

## 简介
管理系统运行所需的各项环境变量与基础配置。

## 功能字段与函数
- `Config`: 配置集合，支持从 `.env` (KEY=VALUE) 或 JSON 格式加载。
  - **基础配置**: `AppName`, `AppMode`, `Port`, `CorsOrigins`。
  - **数据库**: `DBDriver`, `DBDSN` (基于 `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME` 构建)。
  - **极验 (Geetest)**: `GeetestEnabled`, `GeetestID`, `GeetestKey`。
  - **安全**: `JWTSecret`, `AdminPath`, `EnableSwagger`。
  - **邮件 (SMTP)**: `SMTPHost`, `SMTPPort`, `SMTPUser`, `SMTPPass`, `SMTPSSL`, `SystemEmail`。
  - **前端**: `FrontendURL`。
  - **定时任务**: `CleanupIntervalMinutes`（验证码清理间隔，单位分钟，默认 10）。
- `InitConfig`: 初始化函数，优先尝试加载 JSON 格式的 `.env`，失败则回退到标准 `godotenv` 加载。
- `GlobalConfig`: 全局配置单例。

## 环境变量说明 (Key Mapping)
系统兼容以下环境变量（以 JSON 键名为准）：
- `db_host`, `db_port`, `db_user`, `db_password`, `db_name`
- `geetest_enabled`, `geetest_captcha_id`, `geetest_captcha_key`
- `smtp_host`, `smtp_username`, `smtp_password`, `smtp_ssl_type` (填写 `ssl` 开启)
- `system_email_address`, `system_email_name`
- `frontend_url`
- `cleanup_interval_minutes`

## 规范
- 默认值应在代码中明确定义。
- 关键密钥禁止硬编码。
