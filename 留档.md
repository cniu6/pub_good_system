## 🎯 项目简介

FST (Full Stack Template) 是一个基于 Go (Gin) 和 Vue 3 的全栈后台管理系统模板，采用前后端分离架构，支持插件化扩展。

### 技术栈

- **后端**: Go + Gin + MySQL + JWT + Swagger
- **前端**: Vue 3 + TypeScript + Vite + Naive UI + Alova
- **构建**: 支持 Windows/Linux, amd64/arm64

## 🚀 快速开始

### 环境要求

- Go 1.24+
- Node.js 18+ & pnpm
- MySQL 8.0+

### 安装依赖

```bash
# 安装 swag (Swagger 生成工具)
go install github.com/swaggo/swag/cmd/swag@latest

# 安装前端依赖
cd frontend && pnpm install
```

### 启动开发环境

```bash
# 方式1: 直接运行 (推荐) - Swagger 自动更新
go run ./backend/cmd/main.go

# 方式2: 使用脚本
./dev.bat          # Windows
./dev.ps1          # PowerShell

# 方式3: 热重载 (需安装 air)
go install github.com/cosmtrek/air@latest
air
```

### 生产构建

```bash
# Windows
./build.bat

# 构建产物在 build/ 目录
```

## 📚 核心功能

### 1. 用户认证

- JWT Token 认证 (双Token机制)
- 支持用户名/邮箱登录
- 注册验证码
- 密码重置
- 登录失败锁定

📖 **文档**: [doc/JWT认证.md](doc/JWT认证.md)

### 2. 邮件系统

- SMTP 邮件发送
- 多语言邮件模板
- 邮件发送日志
- 支持 SSL/TLS
- 后台可视化管理模板

📖 **文档**: [doc/邮件系统.md](doc/邮件系统.md)

### 3. 插件系统 ✨全自动

- **自动发现**: 扫描 `backend/app/plugins/` 目录
- **自动注册**: 通过 `init()` 自动注册
- **自动文档**: Swagger 自动包含插件 API
- 完整的生命周期管理
- 优先级加载和依赖解析
- 优雅关闭

📖 **文档**: [doc/插件系统.md](doc/插件系统.md)

#### 创建新插件

```go
// backend/app/plugins/myplugin/plugin.go
package myplugin

import (
    "fst/backend/app/plugins"
    "fst/backend/pkg/pluginregistry"
    "github.com/gin-gonic/gin"
)

func init() {
    pluginregistry.Register(NewPlugin())  // 自动注册
}

type MyPlugin struct {
    plugins.BasePlugin
}

func NewPlugin() plugins.Plugin {
    return &MyPlugin{
        BasePlugin: plugins.NewBasePlugin("my-plugin", "1.0.0", "我的插件"),
    }
}

func (p *MyPlugin) RegisterRoutes(router *gin.RouterGroup) {
    router.GET("/my-plugin/hello", p.Hello)
}

// @Summary 测试接口
// @Tags Plugin-MyPlugin
// @Success 200 {object} utils.Response
// @Router /api/v1/my-plugin/hello [get]
func (p *MyPlugin) Hello(c *gin.Context) {
    utils.Success(c, gin.H{"message": "Hello!"})
}
```

**重启服务即可，无需任何手动配置！**

### 4. Swagger 文档 ✨自动更新

- 启动时自动检测代码变化
- 自动重新生成 API 文档
- 自动包含插件 API
- 支持 Bearer Token 认证

访问地址: `http://localhost:8080/swagger/index.html`

### 5. 中间件

| 中间件 | 文件 | 功能 |
|--------|------|------|
| JWT认证 | `middleware/auth.go` | Token验证、用户信息注入 |

| 操作日志 | `middleware/operation_log.go` | 记录用户操作 |
| 请求日志 | `middleware/logger.go` | 请求访问日志 |
| 限流 | `middleware/ratelimit.go` | 接口防刷限流 |

### 6. 服务层

|------|------|------|
| 用户服务 | `services/user_service.go` | 用户CRUD、状态管理 |
| 认证服务 | `services/auth_service.go` | 登录注册验证逻辑 |
| 邮件服务 | `services/email_service.go` | 邮件发送、模板渲染 |
| 清理服务 | `services/cleanup_service.go` | 定时清理过期数据 |

### 7. 数据库

- 用户管理
- 验证码管理
- 邮件模板
- 邮件日志
- 操作日志

📖 **文档**: [doc/数据库模型.md](doc/数据库模型.md)

## 🛠️ 常用命令

```bash

go run ./backend/cmd/main.go

# 生产模式 (跳过 Swagger 更新)
set GO_ENV=production
```

## ⚙️ 环境变量

### 必需配置

```env
# 数据库
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=fst_platform

# JWT
JWT_SECRET=your-secret-key
JWT_ACCESS_EXPIRE=7200
JWT_REFRESH_EXPIRE=604800
```

### 可选配置

```env
# SMTP
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=your-email@gmail.com
SMTP_PASSWORD=your-app-password

# 极验
GEETEST_ENABLED=false
GEETEST_ID=your-geetest-id
GEETEST_KEY=your-geetest-key

# Swagger
ENABLE_SWAGGER=true

# 自动更新控制
GO_ENV=development    # production 跳过自动更新
SKIP_AUTO_SWAGGER=false

# 管理后台路径
ADMIN_PATH=/admin
```

## 🎯 API 架构

### 控制器分层

```
backend/app/controllers/
├── public/     # 公共接口（无需登录）
│   └── auth_controller.go    # 登录/注册/密码重置
├── user/       # 用户接口（需要登录）
│   └── profile_controller.go # 个人信息/修改密码
└── admin/      # 管理接口（需要管理员权限）
    ├── user_controller.go    # 用户管理
    ├── log_controller.go     # 操作日志
    └── email_template_controller.go # 邮件模板
```

### 路由结构

```
/api/v1/
├── public/              # 公共接口
│   ├── POST /login      # 登录
│   ├── POST /register   # 注册
│   ├── POST /forgot-password  # 忘记密码
│   ├── POST /reset-password   # 重置密码
│   └── POST /refresh-token    # 刷新Token
├── user/                # 用户接口（需登录）
│   ├── GET /profile     # 获取个人信息
│   ├── PUT /profile     # 更新个人信息
│   ├── PUT /password    # 修改密码
│   └── GET /routes      # 获取用户路由
├── admin/               # 管理接口（需管理员）
│   ├── GET /dashboard   # 仪表盘
│   ├── /users           # 用户管理 CRUD
│   ├── /logs            # 操作日志
│   └── /email-templates # 邮件模板管理
└── */                   # 插件接口（自动注册）
```
