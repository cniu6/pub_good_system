# 支付订单系统

## 一、系统概述

支付订单系统基于**易支付（Epay）** 协议实现，支持多通道、多支付方式的在线余额充值。核心特性：

- 多通道支付：可配置多个易支付网关，每个通道独立的商户ID/密钥/费率
- 支持支付宝、微信、QQ钱包等支付方式
- 手续费计算：支持「加收」和「包含」两种模式
- 订单自动过期取消
- 异步回调 + 同步跳转双重确认
- 事务级到账处理，保证幂等性和原子性
- 管理员手动补单/取消订单

## 二、关键配置（必须先完成）

### 2.1 系统设置 → 基本设置

在管理后台 → 系统配置 → **基本设置** 中配置以下两项（格式 `http://xxx.com`，结尾不加 `/`）：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **前端地址** (`frontend_url`) | 前端访问地址，支付完成后用户浏览器跳转回此地址 | `http://example.com` |
| **后端API地址** (`backend_api_url`) | 后端API外网可达地址，易支付平台通过此地址发送异步回调通知 | `http://api.example.com` |

> **重要**：创建订单前必须同时配置 `frontend_url` 和 `backend_api_url`，否则会报错，且支付完成后无法正确回跳。

### 2.2 系统设置 → 支付设置

| 配置项 | 说明 |
|--------|------|
| 支付功能开关 | 全局开关，关闭后用户无法充值 |
| 易支付通道开关 | 易支付协议通道总开关 |

### 2.3 管理后台 → 支付通道管理

每个通道独立配置：

| 字段 | 说明 |
|------|------|
| `name` | 通道显示名称（如「支付宝-通道A」） |
| `type` | 通道类型，目前仅支持 `epay` |
| `pay_type` | 支付方式：`alipay` / `wxpay` / `qqpay` |
| `api_url` | 易支付网关地址（如 `https://pay.example.com`） |
| `pid` | 商户ID |
| `key` | 商户通信密钥 |
| `fee_rate` | 手续费率（0-100，百分比） |
| `fee_mode` | `add` = 在充值金额上加收；`include` = 从充值金额中扣除 |
| `min_amount` / `max_amount` | 单笔充值金额范围 |
| `min_level` | 最低用户等级限制（0=不限） |
| `notify_url` | 自定义回调地址（留空则使用全局 `backend_api_url`） |
| `status` | 0=禁用 1=启用 |

## 三、支付流程详解

### 3.1 完整时序图

```
用户(前端)          后端API              易支付平台           数据库
  │                  │                    │                  │
  │ 1.选择通道+金额  │                    │                  │
  │ POST /create ──→│                    │                  │
  │                  │ 2.读取 backend_api_url               │
  │                  │←─────────────────────────────────────│
  │                  │ 3.校验(开关/通道/金额/等级/防刷)       │
  │                  │ 4.计算手续费                          │
  │                  │ 5.创建订单(status=0) ────────────────→│
  │                  │ 6.调用易支付API ────→│                │
  │                  │←── 返回支付链接 ────│                  │
  │←── pay_url ─────│                    │                  │
  │                  │                    │                  │
  │ 7.跳转到支付页面 │                    │                  │
  │ ─────────────────────────────────────→│                  │
  │                  │                    │                  │
  │          (用户完成支付)                │                  │
  │                  │                    │                  │
  │                  │←─ 8.异步通知(POST) ─│                  │
  │                  │   notify_url       │                  │
  │                  │ 9.验签+金额校验     │                  │
  │                  │ 10.事务:改状态+加余额+记日志 ────────→│
  │                  │──→ 返回"SUCCESS" ──→│                  │
  │                  │                    │                  │
  │←──── 9.同步跳转(GET) ────────────────│                  │
  │      return_url → 后端               │                  │
  │                  │ 10.读取 frontend_url                  │
  │                  │←─────────────────────────────────────│
  │←── 302 重定向到前端充值页              │                  │
  │    ?result=success&order_no=xxx       │                  │
  │                  │                    │                  │
  │ 11.前端轮询状态   │                    │                  │
  │ GET /orders/:id/status ──→│           │                  │
  │←── {status:1} ──│                    │                  │
```

### 3.2 URL 使用说明

创建订单时，后端从数据库 `system_settings` 表读取 `backend_api_url`，拼接出两个回调地址：

| 回调地址 | 拼接规则 | 用途 |
|----------|----------|------|
| `notifyURL` | `{backend_api_url}/api/v1/public/payment/notify` | 易支付服务器 → 后端（异步通知，服务器对服务器） |
| `returnURL` | `{backend_api_url}/api/v1/public/payment/return` | 易支付平台 → 后端（同步跳转，用户浏览器经过后端中转） |

同步跳转 `Return` 处理器收到请求后，从数据库读取 `frontend_url`，将用户 302 重定向到前端页面：

| 场景 | 跳转地址 |
|------|----------|
| 支付成功 | `{frontend_url}/#/user/recharge?result=success&order_no=xxx` |
| 等待确认 | `{frontend_url}/#/user/recharge?result=pending&order_no=xxx` |
| 验签失败 | `{frontend_url}/#/user/recharge?result=error&msg=invalid_callback` |

## 四、API 接口一览

### 4.1 用户端（需登录，Bearer Token）

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/user/payment/gateways` | 获取可用支付通道列表（敏感信息已隐藏） |
| `POST` | `/api/v1/user/payment/create` | 创建充值订单，返回 `pay_url` 供前端跳转 |
| `GET` | `/api/v1/user/payment/orders` | 获取我的订单列表（支持分页、状态筛选） |
| `GET` | `/api/v1/user/payment/orders/:id` | 获取订单详情 |
| `GET` | `/api/v1/user/payment/orders/:id/status` | 轮询订单支付状态 |

#### 创建订单请求示例

```json
POST /api/v1/user/payment/create
{
  "gateway_id": 1,
  "amount": 50.00,
  "subject": "余额充值"
}
```

#### 创建订单响应示例

```json
{
  "code": 200,
  "data": {
    "order_no": "P202603081930000100001234",
    "pay_url": "https://pay.example.com/cashier.php?trade_no=xxx",
    "amount": 50.00,
    "fee": 0.00,
    "pay_amount": 50.00,
    "expire_at": 1741443000,
    "gateway_name": "支付宝-通道A",
    "payment_type": "alipay"
  }
}
```

### 4.2 公共接口（无需登录）

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST/GET` | `/api/v1/public/payment/notify` | 易支付异步通知回调，返回 `SUCCESS` 或 `FAIL` |
| `GET` | `/api/v1/public/payment/return` | 易支付同步跳转，302 重定向到前端 |

### 4.3 管理端（需管理员权限）

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/admin/payment/orders` | 获取所有订单（支持搜索、筛选） |
| `POST` | `/api/v1/admin/payment/orders/:id/complete` | 手动补单（到账） |
| `POST` | `/api/v1/admin/payment/orders/:id/cancel` | 取消订单 |
| `DELETE` | `/api/v1/admin/payment/orders/:id` | 删除订单 |
| `GET` | `/api/v1/admin/payment/stats` | 支付统计数据 |
| `GET` | `/api/v1/admin/payment/gateways` | 管理支付通道列表 |
| `POST` | `/api/v1/admin/payment/gateways` | 创建支付通道 |
| `PUT` | `/api/v1/admin/payment/gateways/:id` | 更新支付通道 |
| `DELETE` | `/api/v1/admin/payment/gateways/:id` | 删除支付通道 |

## 六、手续费计算

### 6.1 加收模式 (`fee_mode = "add"`)

用户多付手续费，到账金额 = 充值金额。

```
充值金额 = 50 元，费率 = 3%
手续费 = 50 × 3% = 1.50 元
实际支付 = 50 + 1.50 = 51.50 元
到账金额 = 50 元
```

### 6.2 包含模式 (`fee_mode = "include"`)

手续费从充值金额中扣除，到账金额减少。

```
充值金额 = 50 元，费率 = 3%
手续费 = 50 × 3% = 1.50 元
实际支付 = 50 元
到账金额 = 50 - 1.50 = 48.50 元
```

## 七、安全机制

| 机制 | 说明 |
|------|------|
| **MD5 签名验证** | 所有回调参数按 ASCII 排序拼接后加密钥取 MD5，防篡改 |
| **事务处理** | 回调到账使用 `SELECT ... FOR UPDATE` 锁行 + 事务，保证原子性 |
| **幂等性** | 同一订单重复回调只处理一次（检查 `status != pending` 则跳过） |
| **金额校验** | 回调金额与订单 `pay_amount` 比对（允许 ±0.01 误差） |
| **防刷机制** | 单用户最多 10 个未支付订单 |
| **订单过期** | 超过有效期（默认 30 分钟）自动取消 |
| **XSS 防护** | 订单标题经过 XSS 清理 |
| **敏感信息隐藏** | 用户端 API 不返回通道密钥、PID、API 地址等 |
